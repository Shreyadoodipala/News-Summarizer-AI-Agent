# News-Summarizer-AI-Agent

An AI-powered news research agent built with **LangGraph**, **LangChain**, **Groq (Llama 3)**, and **Streamlit**. Give it a topic, and it will search [NewsAPI](https://newsapi.org/), scrape the full text of relevant articles, pick the most relevant ones, and generate bulleted TL;DR summaries — all through an interactive web UI.

## How it works

The agent is a cyclical graph (built with LangGraph) that:

1. **Generates NewsAPI search parameters** from your natural-language query using an LLM (keywords, date range, language, sort order).
2. **Calls the NewsAPI** to retrieve matching article metadata.
3. **Scrapes each article's full text** from its URL.
4. **Loops back to step 1** with loosened search terms if not enough usable articles were found yet (up to a configurable search depth).
5. **Selects the top N most relevant articles** for the query using the LLM.
6. **Summarizes each selected article** into a bulleted TL;DR.
7. **Formats and returns the final results.**

```
START
  └─> generate_newsapi_params
        └─> retrieve_articles_metadata
              └─> retrieve_articles_text
                    ├─(not enough articles & searches left)─> generate_newsapi_params (loop)
                    ├─(enough articles, or out of searches)─> select_top_urls
                    └─(out of searches, no articles found)──> END
select_top_urls
  └─> summarize_articles_parallel
        ├─(summaries produced)─> format_results ─> END
        └─(no articles to summarize)─> END
```

Run `visualize_workflow.py` (included in this repo) to generate this diagram as Mermaid, PNG, or ASCII directly from the compiled graph.

## Prerequisites

- Python 3.9+
- A [Groq API key](https://console.groq.com/) (used to run Llama 3 for parameter generation, article selection, and summarization)
- A [NewsAPI key](https://newsapi.org/register) (free tier available, used to search for articles)

## Setup

1. **Clone the repository and install dependencies:**

   ```bash
   pip install -r requirements.txt
   ```

2. **Create a `.env` file** in the project root with your API keys:

   ```env
   GROQ_API_KEY=your_groq_api_key_here
   NEWSAPI_KEY=your_newsapi_key_here
   ```

3. **Run the Streamlit app:**

   ```bash
   streamlit run app.py
   ```

   This opens a local web UI where you can enter a query and configure search options.

## Usage

In the Streamlit UI:

- **Enter your query** — a topic or question you want news coverage on.
- **Number of Searches (Search depth)** — how many times the agent may query NewsAPI (loosening search terms each time) if not enough articles are found.
- **Number of Articles to summarize** — how many top articles to summarize in the final output.
- **Search in** — which article fields (title, content, description) NewsAPI should match your query against.
- **Language** — the language of articles to search for.
- **Sort By** — `relevancy`, `popularity`, or `publishedAt`.

Click **Search News** and the agent will search, scrape, select, and summarize articles, then display bulleted TL;DR summaries with sources.

## Visualizing the workflow

To inspect the graph structure without running the full agent:

```bash
python visualize_workflow.py
```

This will:
- Print the list of graph nodes
- Save a Mermaid diagram (`workflow_diagram.mmd`) you can paste into [mermaid.live](https://mermaid.live)
- Attempt to render `workflow_diagram.png` (via mermaid.ink or local Graphviz)
- Fall back to an ASCII diagram (`workflow_ascii.txt`) if PNG rendering isn't available

## Notes & limitations

- Article scraping uses `requests` + `BeautifulSoup` with a generic `User-Agent` header; some sites (paywalled, JS-rendered, or those that block scrapers) may fail or return partial text.
- The agent is capped at a configurable number of search iterations to control API usage and runtime.
- LLM outputs (search parameters, article selection, summaries) are generated by Llama 3 8B via Groq, and are parsed/repaired with `OutputFixingParser`; results may occasionally require re-running if parsing fails.
